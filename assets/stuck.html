Help Stuck Students

Use this page during class when students cannot move forward. Start with the smallest fix that lets them keep learning, then return to the root cause later.

Notebook Does Not Run

  1. Save the notebook.
  2. Use Kernel > Restart Kernel.
  3. Run cells from the top again.
  4. Check whether the first error is an import, path, syntax, or data problem.

Most notebook errors after many edits come from hidden kernel state. Restarting gives the student a clean, repeatable result.

Wrong Notebook Environment

  1. For individual practice, use Student practice - local Jupyter Notebook.
  2. For centralized class control, use Teacher controlled - JupyterHub.
  3. If students cannot see the same files, confirm they opened the cloned assignment folder.
  4. If the hub login appears, ask the teacher to confirm the JupyterHub URL and account access.

Common Notebook Errors

ModuleNotFoundError

Install requirements, then restart the kernel.

FileNotFoundError

Check the notebook folder and dataset path. Use relative paths when possible.

NameError

Run earlier cells first, or check spelling of variables and functions.

Kernel died

Restart the kernel. If it repeats, reduce memory use or check package versions.

Pull Latest Starter Code

  1. Save current work.
  2. Open a terminal in the assignment folder.
  3. Run git status to see local changes.
  4. Run git pull to get instructor updates.

If Git reports conflicts, ask the student to copy their answer cells into a backup text file before resolving anything.

Submit Again

  1. Run the notebook from top to bottom.
  2. Save all files.
  3. In the app, choose Assignment > Upload assignment.
  4. Paste a GitHub token when asked.
  5. Wait for the pushed status badge.
