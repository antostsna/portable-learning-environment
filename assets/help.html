Portable Learning Environment Help

Use this page when you need to start notebooks, clone assignments, submit work, or fix common classroom setup problems.

Start a Lesson

  1. Click Pre-flight check before class to confirm Python, Git, Jupyter, packages, and Git identity.
  2. Click Start lesson on the home screen or Assignment menu.
  3. Choose the course type: General lecture, Programming language, Digital image processing, AI / machine learning, or General lab.
  4. Choose the delivery mode: local Jupyter Notebook for student practice, or teacher-controlled JupyterHub.
  5. Paste the GitHub Classroom repository URL.
  6. Choose the work folder for student files.
  7. Review or edit the checklist, then click Start.
  8. The app opens the selected notebook environment, clones the assignment when provided, opens the work area, and shows the checklist.

Choose the Right Delivery

Student practice - local Jupyter Notebook

Use this when students work independently on their own computer. The app starts a local notebook server only after the lesson begins.

Teacher controlled - JupyterHub

Use this when the instructor or school manages the notebook server. Enter the JupyterHub URL, then students work through the controlled hub environment.

Start a Notebook Session

  1. Click Open JupyterHub on the home screen or toolbar.
  2. Wait a few seconds while the notebook server starts.
  3. The notebook page should open inside this app, not in an external browser.
  4. Open an existing notebook or create a new notebook from the Jupyter file list.

If your class uses a hosted JupyterHub, set PLE_JUPYTER_URL to the hub URL before starting the app.

Use GitHub Classroom

  1. Accept the assignment link from your instructor.
  2. In this app, choose Assignment > Start lesson or Clone assignment.
  3. Select a working folder, paste the GitHub repository URL, then click Clone.
  4. Open the cloned folder in JupyterHub and work on the notebook files.
  5. When finished, choose Assignment > Upload assignment and paste a GitHub token.

Common Problems

JupyterHub does not open

Wait 10-20 seconds and click Open JupyterHub again. If it still fails, check that Jupyter is installed with pip install -r requirements.txt.

A page opens in Chrome or Edge

The app starts local Jupyter with --no-browser. If an external browser still opens, close it and use the JupyterHub tab inside this app.

Git clone fails

Check that Git is installed, the repository URL is correct, and you have permission to access the assignment repository.

Upload to GitHub fails

GitHub no longer accepts normal account passwords for many operations. Use a personal access token when authentication is required.

Assignment Status Badges

  • Cloned: the GitHub Classroom repository exists on this computer.
  • Opened: the assignment folder was opened in JupyterHub.
  • Submitted: the student marked the work ready for upload.
  • Pushed: the latest commit was pushed to GitHub.

Submission Quality Check

Before upload, the app checks whether the selected folder is a Git repository, Git identity is configured, notebooks are readable, empty code cells exist, outputs or artifacts are present, and Git has changes to submit.

Failed checks should be fixed before upload. Warnings can be accepted when the teacher confirms they are expected for the assignment.

For Students

  • Run every notebook cell from top to bottom before submitting.
  • Save your notebook before upload.
  • Do not rename required files unless your instructor says to.
  • If code breaks, copy the full error message before asking for help.
  • Submit early once, then improve and submit again if your class allows it.

For Teachers

  • Put the learning goal and expected output at the top of each notebook.
  • Use small checkpoints so students know if they are still on track.
  • Provide starter code for setup-heavy tasks and leave the concept work to students.
  • Ask students to commit meaningful milestones, not only final answers.
  • Include a short reflection question to reveal misconceptions.

Helpful Commands

  • Install requirements: pip install -r requirements.txt
  • Start the app: python main.py
  • Check Git: git --version
  • Check Python: python --version

On Windows, if installation fails with a long-path error, use a shorter project path such as C:\ple or enable Windows Long Path support.
